概述

邻接+关联

  • 顶点集V（有限集），边集E

  • 存在两两关系：以连边来表示

  • 邻接关系：存在连边的任何两个点，这两个点被称为彼此邻接（V~V）

  • 关联关系：参与定义邻接关系的每一个顶点，与这个邻接关系之间的关系，被称为关联关系（V~E）

  • 序列结构和树结构是图的一种特例，即图更为一般化

  • 任何两个节点之间都允许存在邻接关系

无向+有向

  • a：无向图

    • 若邻接顶点u和v的次序无所谓，则（u，v）为无向边
    • 所有均无方向的图，即无向图

  • b：有向图

    • 有向图中均为有向边，u、v分别称作边（u，v）的尾和头
  • c：混合图
    • 不仅有无向边，也有有向边

路径+环路

  • 简单路径：在一条通路中不含重复节点
    • 简单环路：路径的终点和起点重合
  • 欧拉环路：经过所有的边一次，而且恰好一次的环路
  • 哈密尔顿环路：经过所有的顶点一次，而且恰好一次的环路

Graph模板类

邻接矩阵与关联矩阵

邻接矩阵

  • 图有n个顶点，就构造一个$n*n$的矩阵
  • 第i行第j列的那个单元中，存储顶点i和顶点j的邻接关系。比如1代表邻接，0代表不邻接
  • 如果是无向图，那么对应的邻接矩阵应该是对称的。也就是说第i行第j列的元素和第j行第i列的元素雷同。在对角线上的元素，为自环。
  • 如果是带权图，那么矩阵单元内只需要存储整型的$w$。

关联矩阵

  • 图有n个顶点，e条边，就构造一个n行e列的矩阵。
  • 矩阵单元内存储着对应的点和边之间的关联关系。如果存在关联关系，就存1；否则就存0
  • 关联矩阵的每一列中，应该恰好只有两个单元的数值为1，其余的都是0

总结

复杂度

  • 判断两点之间是否存在联边：$O(1)$
  • 获取顶点的（出/入）度数：$O(1)$
  • 添加、删除边后更新度数：$O(1)$

优点

  • 直观，易于理解和实现
  • 适用范围广泛：
    • digraph
    • network
    • cyclic
    • dense graph
  • 扩展性：得益于Vector良好的空间控制策略，空间溢出等情况可以”透明地“予以处理

缺点

平面图：可嵌入于平面的图，相邻的边不会交叉

欧拉公式中，v：顶点数；e：边数；f：区域面片的总数；c：连通域的总数。

对于平面图来说，它的边数不会超过顶点数n。但是一个邻接矩阵的空间占用达到了$n^2$。此时的空间利用率约等于$1/n$，空间利用率极低。

广度优先搜索

策略

  • 按照与起点S的距离，将所有的顶点划分为等价类

  • 在同一等价类的内部，各顶点的边都不会被采纳

  • 只有连接相邻等价类的某些边，才会被采纳

  • 即避开环路，将原先的图构成一个极大的无环图，也就是树。这棵树中涵盖了图的所有顶点，所以也叫支撑树

  • 这样的遍历过程类似于树的层次遍历
  • 每个顶点与s之间的那条通路，恰好就是在原图中，这两个顶点之间的那条最短通路

算法

算法细节

在算法开始时，将除了开始顶点的其余顶点的状态设置UNDISCOVERED，开始顶点的状态设置DISCOVERED并入队。为每条边的状态设置为UNDETERMINED。

当前顶点v找到的邻居u的状态：

  • 如果为UNDISCOVERED，即未被发现的状态，那么将u的状态设置为DISCOVERED，再将u入队。并且采纳v和u的边，同时将边的状态设置为TREE。
    • TREE 边的数量总是等于顶点数减去连通分量的数量

  • 如果为DISCOVERED，即已经被发现的状态，说明顶点u此时在队列中，或者已出队。那么将v和u的边的状态设置为CROSS。

  • 除了开始顶点，其余顶点皆会通过某种处理方式忽略掉父亲

  • 整个遍历的最终产物是一棵遍历支撑树
  • 顶点v所属的那个连通域，可以被悉数的遍历

复杂度

由于邻接矩阵采用的Vector在物理上连续，所以访问邻接矩阵中的元素实际上是在高速缓存中，速度会非常的快。因为这个原因，理论上分析的复杂度$O(n^2)$会变为$O(n+e)$。

算法细节

当前顶点v找到的邻居u的状态：

  • UNDISCOVERED：意味着可以从当前顶点u继续拓展，将v通往u的这条边引入到遍历树中，即设置这条边的状态为TREE，将顶点u的父亲设置为v，并将控制权交给顶点u。然后按照策略递归的执行DFS。
  • DISCOVERED：将v到u的边的状态设置为BACKWARD，即回向边。
    • 回向边：试图从一个后代去回连到它的祖先。一旦发现了一条回向边，就说明至少出现了一条回路。
  • default：此时代表u已经被访问完毕，也就是处于最终的VISITED状态，此时判断v和u的dTime大小
    • 如果是v更早被发现，就将v到u的边设置为FORWARD，也就是前向边
      • 前向边：由遍历树中的祖先节点向前，指向它的后代
    • 如果是u更早被发现，就将v到u的边设置为CROSS，也就是交叉边或跨越边。
      • 跨越边：他们的祖先没有直系血缘关系，即他们是姑表亲

准则

BCC

  • 图中的一个点摘除之后，图中的连通域的数量会有所增加，那么这个点就是关节点。

  • 任何一张连通的无向图，都存在着若干个关键点。而且以这些关键点为界，可以把图分割成几个若干个自洽的双连通的部分。

叶子与非叶子

DFS所构成的连通图中：

  • 叶子不会是关节点

  • 根不一定是关节点，因为根在DFS中是随机选的

    • 如果根有1度，那么它不是关节点
    • 如果根的度数大于1度，那么它是关节点
  • 内部节点V的子树中，某一个孩子向上通过回边能达到的更高的祖先：
    • 比V的辈分大，那么V不是关节点
    • 如果就是V，那么V就是关节点

所以可以同过一个指标，来判断更高的祖先，这个指标为$hca(v)$
