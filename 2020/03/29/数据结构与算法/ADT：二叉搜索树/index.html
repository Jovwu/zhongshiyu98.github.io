二叉搜索树

BST：Binary Search Tree

性质

循关键码访问

在BST中，数据项之间，依照各自的关键码彼此区分。关键码之间支持：

  • 大小比较
  • 相等比较
词条

数据集合中的数据项，统一地表示和实现为词条entry形式。

组成单位

  • 节点
    • 词条：在节点内
      • 关键码：在词条内，且唯一

特征

顺序性
  • 顺序性：任意节点均不小于/不大于其左/右后代

  • 单个节点的二叉树必然是BST
  • 只要满足局部顺序性的单分支二叉树，也是BST

  1. 不是BST，因为不是二叉树
  2. 不是BST，因为词条等于3的节点的右后代中，有一个词条等于2的节点，不满足局部顺序性。
  3. 是BST
  4. 是BST
  5. 是BST

顺序性虽然只是对局部特征的刻画，但由此却可导出某种全局特征：单调性

单调性

单调性：BST的中序遍历序列，必然单调非降。这个性质也是BST的充要条件。

  • 一棵树中所有节点的垂直投影，就是这棵数的中序遍历序列。

  • 只要这个序列是单调变化的，那么这棵树必然是BST。

  • BST在微观上处处满足顺序性，在宏观上，整体满足单调性。

接口

由BinTree类派生而来

BST：查找

实例

减而治之：从根节点出发，逐步地缩小查找范围，直到发现目标（成功），或查找范围缩小至空树（失败）。

对照中序遍历序列可以看出，整个过程可视作是在仿效有序向量的二分查找。

实现

这个算法每递归一次，当前节点V就会下降一层，因此这个算法在最坏情况下的递归深度不会超过树高。所以这个算法所对应的时间复杂度为$O(h)$。

在每一次深入递归之前，都会修改hot变量。所以hot变量表示刚刚访问的那个非空的节点。

BST：删除

算法框架

元素删除

情况一

若x将要被删除时：

  • x只有左子树
  • x只有右子树
  • x左右子树都没有

若被删除的元素的某一子树为空，那么可将其替换为另一子树

情况二

若此时被删除的节点x有左右孩子：

  • x的直接后继是x的右孩子
  • x的直接后继是x的右子树中的后代

处理办法：

  1. 先找到x的直接后继w，并互换数据域，相当于等效地将待删除节点转移位置，而且这个位置只有一个分支，且这个分支只可能是右孩子。原因是在BST中x的直接后继是沿着右子树左侧链下行至最后一个，即x在中序遍历下的直接后继
  2. 并将这个右孩子跟新的父亲之间进行双向连接。

算法实现

复杂度

主要取决于removeAt中第二种情况的succ()，所以不会超过$O(h)$。

BST：平衡与等价

极端退化

BST的极端退化成单链，此时的树高等于节点数-1。即最坏情况下的复杂度为$O(n)$。

平均高度

随机生成

当关键码总数为n时，可能的生成序列数量为$n!$。$n!$棵BST平均高度为$log_2{n}$。

但是用这种方式会有重复，所以不可靠。不同的关键码序列，有可能生成同一棵BST。比如2,1,3和2,3,1。

推广：中位数或者是越接近中位数的关键码越早的被插入，这棵BST的高度也会更低。

随机组成

把n个关键码视作n个互异的积木，可能的生成序列数量为$catalan(n)$，$catalan(n)$棵BST平均高度为$\sqrt{n}$。

平衡度

理想平衡

实际应用中，理想平衡可遇不可求

适度平衡

在理想平衡上放松标准：高度渐进地不超过$O(log_2{n})$，即可称作适度平衡

等价变换

性质

中序遍历序列的歧义性：在两棵结构不同的BST中，它们的中序遍历序列有可能是完全雷同的。

相互等价的BST：拓扑结构不同，但中序遍历序列却相同的任何一对BST。它们的特点有：

  • 上下可变：联接关系不尽相同，承袭关系可能颠倒。（垂直方向有一定的自由度）
  • 左右不乱：中序遍历序列完全一致，全局单调非降。

所以可以通过以上两点性质，对失衡的BBST重新转换成平衡的BBST

变换

zig变换

在此局部围绕着节点V做顺时针的旋转

zag变换

在此局部围绕着节点V做逆时针的旋转
