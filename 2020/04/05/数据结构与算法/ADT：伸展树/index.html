伸展树

逐层伸展

思路

自适应调整：借助程序的局部性，并且按照链表的思路，将最近访问的元素挪到最前面。

策略

  • 节点v一旦被访问，随即转移至树根
  • 自下而上，逐层单旋

反复的向上转移，直到到达树根

实例

访问333

经过一系列的zig和zag旋转

最坏情况

旋转次数呈周期性的算术级数演变：每一周期累计$\Omega(n^2)$，分摊$\Omega(n)$。

它已经退化成了最原始的线性序列！

双层伸展

思路

  • 向上追溯两层，而非一层
  • 反复考察祖孙三代：g = parent(p)，p = parent(v)，v
  • 根据它们的相对位置，经两次旋转使得v上升两层，成为（子）树根。

zig-zag/zag-zig

观察发现：

  • 与AVL树双旋完全等效
  • 与逐层伸展别无二致

zig-zig/zag-zag

将原本的zig(p)->zig(g)的顺序变为zig(g)->zig(p)的顺序。

zig/zag

若是v只有父亲，没有祖父：

  • 此时必有parent(v) == root(T)，且每轮调整中，这种情况至多（在最后）出现一次。
  • 视具体形态，做单次旋转zig(r)或zag(r)

路径折叠

  • 折叠效果：一旦访问坏节点，对应路径的长度将随即减半

  • 这种折叠效果，具有对坏节点的修复作用

  • 最坏情况不致持续发生
  • 单趟伸展操作，分摊$O(log_2{n})$时间

伸展算法

查找算法

  • 伸展树的查找操作，与常规BST::search()不同，很可能会改变树的拓扑结构，不再属于静态操作。

插入算法

  • 实际上，查找（失败）之后，_hot即是根节点。所以_hot为插入节点的左/右孩子，插入节点最终将变为树根。

删除算法

  • 通过Splay::search()查找（成功）之后，目标节点即是树根，所以可以直接在树根附近完成对目标节点的删除

  • 删除目标节点后，找到右子树中的最小者或左子树中的最大者m，作为新的树根

  • 在此之后，作为树根的节点m，是与此前被删除的节点非常临近的一个，所以在此后局部性将可以继续得到充分的利用。

综合评价

优点

  • 无需记录节点高度或平衡因子，所以编程实现简单易行（优于AVL树）
  • 分摊复杂度$O(log_2{n})$，与AVL树相当
  • 局部性强，缓存命中率极高（即$k<<n<<m$）
    • 效率甚至可以更高：自适应的$O(log_2{k})$
    • 任何连续的m次查找，都可以在$O(mlog_2{k}+nlog_2{n})$时间内完成

缺点

  • 仍不能保证单次最坏情况出现
  • 不适用于对效率敏感的场合（比如手术器械）
