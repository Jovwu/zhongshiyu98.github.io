红黑树

结构

规则

红黑树是由红、黑两类节点组成的BST，统一增设外部节点NULL，使之成为真二叉树。其中：

  • 树根：必为黑色
  • 外部节点：均为黑色
  • 其余节点：若为红，则只能有黑孩子。即红节点的父子皆黑（这条规则旨在控制红黑树的深度）
  • 外部节点到根：途中黑节点数目相等（这条规则旨在控制红黑树的平衡性）

实例

提升变换

  • 将每一个红色的节点都向上提升至与它的父亲平齐。提升后，两个红色节点一定不会紧邻，中间一定有一个黑色节点，也就是它们此前的父亲。

(2,4)树 == 红黑树

红黑树提升变换：

  • 提升各红节点，使之与其（黑）父亲等高，于是每棵红黑树，都对应于一棵$(2,4)$-树。
  • 将黑节点与其红孩子视作（关键码并合并为）超级节点
  • 无非四种组合，分别对应于4阶B-树的一类内部节点

红黑树$\in$BBST

  • 由等价性，既然B-树是平衡的，红黑树自然也是。

  • 定理：包含n个内部节点（n+1个外部节点）的红黑树T，高度$h=O(log_2{n})$

  • 若：T高度为h，黑高度为H，则$h=R+H\le2H$

  • 若：T所对应的B-树为$T_B$，则H即是$T_B$的高度。
  • $T_B$的每个节点（超级节点），包含且仅含包含T的一个黑节点。

效果如下：

  • 调整前之所以非法，是因为在某个三叉节点中插入红关键码，使得原黑关键码不再居中。即RRB或BRR，出现相邻的红关键码。
  • 调整之后的效果相当于B-树的拓扑结构不变，但在新的四叉节点中，三个关键码的颜色改为RBR。
  • 虽然红黑树的拓扑结构有所调整，但仅限于局部。更重要的是，这种调整是一蹴而就的，无需任何进一步的调整。因此，就全树的拓扑连接关系变化量而言，必然是不超过$O(1)$。

u->color == R

  • 在B-树中，等效于超级节点发生上溢

修复这种情况下的双红缺陷，可以看作是修复B-树中的上溢。

解决办法：

  • p与u转黑
  • g转红

在B-树中，等效于节点分裂，关键码g上升一层。

在B树角度来看，这种调整过程发生了拓扑结构变化。但是从红黑树的角度来看，除了若干个节点的颜色会发生改变，全树的拓扑连接关系并没有任何变化。也就是说虽然重染色的操作可能会达到$O(log_2{n})$，但是拓扑结构的操作只会$O(1)$。

总结

  • 重构操作对持久化结构而言至关重要
  • 实际上AVL树的插入操作也能达到这要求

过程

  • 首先按照BST常规算法，执行r = removeAt(x,_hot)，此时将删除节点x和替代节点r的颜色分情况讨论：（r可能是个外部节点）
    • 两者中有一个红色：将r染黑即可
    • 两者都是黑色：发生双黑缺陷。因为违反了规则4（即外部节点到根：途中黑节点数目相等）。

双黑缺陷

  • 摘除x并代之以r后，全树黑深度不再统一，原B-树中x所属节点下溢
  • 修复红黑树的双黑缺陷，实际上是修复B-树的下溢
  • 在新树中，需要考察：
    • r的父亲：p = r->parent
    • r的兄弟：s = r == p->lc ? p>rc : p->lc

BB-1：S为黑，且至少有一个红孩子t

这种情况下的修复过程为：

  1. 3+4重构：t、s、p重命名为a、b、c

  2. r保持黑色

  3. a和c染黑

  4. b继承p的原色

此时红黑树中的黑深度保持一致，全局性质得以恢复。（与zig-zag等类似）

从B-树的角度看：

通过关键码的旋转，消除超级节点的下溢。

下溢节点从父亲那借得一个关键码，父亲再向下溢节点的兄弟那里借一个关键码，以填补空缺。

BB-2R：s为黑，且两个孩子均为黑；p为红

这种情况下的处理办法：（此时的s已处于下溢边缘，无法借出关键码）

  1. r保持黑

  2. s转红

  3. p转黑

在对应的B-树中，等效于下溢节点与兄弟合并。合并之前，在p之左或右侧还应有（问号）关键码必为黑色，有且仅有一个。所以失去关键码p后，上层节点并不会发生下溢。

BB-2B：s为黑，且两个孩子均为黑；p为黑

这种情况下的处理方法：

  1. s转红

  2. r与p保持黑

从B树的角度来看，如果下层下溢引发上层下溢，且在极端情况下会传播到树根。这时候的重构可能多达$O(log_2{n})$次。但是从红黑树的角度来说，拓扑结构实际上没有任何的改变，所以还是$O(1)$。

BB-3：s为红（其孩子均为黑）

这种情况下的处理方法：

  1. zag(p)或zig(p)
  2. 红s转黑
  3. 黑p转红

虽然此时黑高度依然异常，但r有了一个新的黑兄弟s`。所以可以转化为之前的三种情况，而且：

  • p已经转红，接下类绝不会是情况BB-2B，而只能是BB-1或BB-2R

总结

  • 每一次删除操作，在每一高度至多只会花费常数时间，即删除操作可以在$O(log_2{n})$​​时间内完成
  • 其中，至多做：
    1. $O(log_2{n})$次重染色
    2. 一次”3+4“重构
    3. 一次单旋
